#1: Time Complexity

3rd February, 2021

Time Complexity basically is the amount of time it takes to run an algorithm. It is generally estimated by counting the number of operations performed by an algorithm.

We generally consider worst-case time complexity(maximum amount of time taken for an algorithm)
Time complexity is generally expressed using big O notation. Example O(N), O(N^2), O(log N), etc. (here, N=input size)

I will try to explain time complexity by a simple example, consider a phone book (phone book is alphabetically sorted) of N page. You want to find the number of your friend.

1st Approach: Start from the first page and check every page for your friend's number. (This algorithm is called Linear Search)
Worst-case (your friend's name starts with Z): You need to check all N pages for your friend's number.
So time complexity is O(N)

2nd Approach (assume your friend's name starts with C): Since the phone book is alphabetically sorted, you know 'C' will be in the first half of the book.
You now need to search in that first half only. Similarly, you also know 'C' will be in the first half of the original first half. Likewise, you can keep dividing your search space in half at each step.
The above algorithm is called Binary Search, whose worst-case time complexity will be O(log N)
